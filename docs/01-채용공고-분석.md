## 3. 신입/주니어 vs 경력 요구사항

### 신입~주니어 (0~3년)에게 요구되는 것

1. **언어 1개 + Spring 기본기** — "한 개 이상의 언어를 자유롭게 사용하여 제품 개발 경험"(29CM)
2. **CS 기초** — "설계, 디버깅, 자료구조 및 알고리즘의 이해"(네이버파이낸셜)
3. **문제 해결 경험의 '구체성'** — 토스페이먼츠 이력서 가이드: "개발 문제 상황을 개선하거나 해결해 본 사항을 최대한 구체적으로". **연차 대신 밀도로 평가**
4. **레거시/기존 코드에 대한 태도** — "재설계를 즐기는 마인드"(29CM), "레거시 분석이 빠르고 개선 의욕"(네이버)
5. **유지보수성 지향** — 심플한 설계, 테스트 코드 습관
6. **사전과제/기술과제 대비** — 토스페이먼츠·NHN은 과제 전형. 실제 구현 능력으로 승부

**신입 문이 실제로 열려 있는 곳**: 토스페이먼츠(3년 이하 공채), KG이니시스(신입·경력 상시), 교보문고(주문/결제 신입 직무), 29CM(신입~4년), 토스(3년 이하 39%)

### 경력 (3년+)에게 요구되는 것 — 포트폴리오로 '모사'할 대상

1. **프로덕션 장애 해결 경험** (토스·카카오페이 필수) → 신입은 실무 장애가 없으므로 **장애 재현 실험으로 대체** (04 문서)
2. **대용량 설계** → **부하테스트 수치로 대체** (05 문서)
3. **MSA 설계 + 장애 복구 시나리오** → 서킷브레이커/폴백/복구 배치 구현으로 대체
4. **DB 심화** (쿼리 최적화, 인덱스 튜닝) → EXPLAIN 전후 비교 리포트로 대체
5. **도메인 지식** (결제 프로세스, 정산 프로세스 이해) → 02·03 문서의 개념을 구현으로 증명

## 4. 공고 → 포트폴리오 요소 매핑 (역산)

공고 문구를 역산하면 신입 포트폴리오는 이 순서로 강력하다:

| 우선순위 | 포트폴리오 요소 | 근거 공고 문구 |
|---|---|---|
| 1 | PG 연동 결제 플로우 end-to-end (승인→취소→부분취소→망취소) | "PG연동"(배민·페이코·크림), "승인시스템"(이니시스) |
| 2 | 멱등성 처리 (멱등키, 재시도, 타임아웃 상태 복구) | 인턴 공고 명시 + 대형사 면접 빈출 |
| 3 | 동시성 제어 (포인트/잔액 차감, 선착순 결제) | 토스페이먼츠 "선착순 결제, 수십배 트래픽" |
| 4 | 정산 배치 (거래 집계→수수료 계산→지급 데이터) | 토스페이먼츠 "오차 없는 정산", 다날 Spring Batch 필수 |
| 5 | 대사 로직 (PG 내역 vs 자체 데이터 비교·보정) | 크림 "대사 시스템 개발", 페이코 "송금 데이터 검증" |
| 6 | 부하테스트 실측 수치 (TPS, p99, 튜닝 전후) | 네이버 "대용량", 카카오페이 "병목 진단" |
| 7 | 이벤트 기반 아키텍처 (Kafka, Outbox, Saga) | Kafka 스택(토스페이먼츠·카카오페이·섹타나인) |
| 8 | 장애 시나리오 문서화 (장애 주입→서킷브레이커→복구) | 카카오페이 "장애 복구 시나리오 구축" |
| 9 | 테스트 코드 + JPA 도메인모델 상태머신 | "JPA 도메인모델 방식"(다날·배민 공통 문구) |
| 10 | 레거시 리팩토링 스토리 (before/after) | 토스페이먼츠 우대사항 그 자체 |
| 11 | 운영 가시성 (Grafana, 결제 실패율 대시보드) | 다날 ELK, 크림 모니터링 |
| 12 | 무중단 배포, AI 도구 활용 워크플로 | 카카오페이 최신 요건 |

> **결론**: (1) PG연동 + (2) 멱등성/동시성 + (4) 정산 배치 + (5) 대사 조합이 "결제 도메인을 이해하고 있다"는 가장 강력한 신호. 여기에 (6) 부하테스트 수치와 (8) 장애 시나리오를 붙이면 토스페이먼츠·카카오페이류 공고의 필수/우대 문구를 거의 전부 커버한다.
